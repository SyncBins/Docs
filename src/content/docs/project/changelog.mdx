Curated release notes — not auto-generated from every commit. During alpha, watch [GitHub Releases on App](https://github.com/SyncBins/App/releases) for tags like `v0.1.0-alpha.1`.

## Unreleased / alpha

SyncBins is **pre-1.0**. The hosted product and docs evolve together. Notable areas in the current codebase:

- Web client: onboarding wizard (create / pair / restore), Inbox + bins, composer, drag/drop, clipboard hint, device pairing, quick-unlock PIN, Settings panes
- Server: Fastify REST + WebSocket, SQLite, local/R2/Azure blobs, single- and multi-tenant modes, Stripe hooks (hosted)
- Integrations: `@syncbins/mcp` MCP tools

This list describes what exists today, not a promise of stability.

## How releases work (maintainers)

Pushing a git tag `v0.1.0-alpha.N` triggers CI to build the Docker image (`ghcr.io/syncbins/syncbins`), push to GHCR, and attach a deploy zip. See [App README → Cut an alpha release](https://github.com/SyncBins/App#cut-an-alpha-release-maintainers).

Self-hosters: [Upgrades](/self-hosting/upgrades/).
