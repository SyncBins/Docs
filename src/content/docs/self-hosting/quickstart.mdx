Self-hosters run **the same server** as syncbins.com — one Node/Fastify process serving the React app, REST API, and WebSocket relay. The web UI is identical; only who runs the box and where blobs live differs.

<Aside type="tip" title="What you'll have when this is done">
A SyncBins instance you control. Metadata in SQLite on disk; encrypted blobs on **local disk by default** (no cloud account required). Optional: point blobs at [Cloudflare R2 or Azure](/self-hosting/storage/) via env vars.
</Aside>

## Pick your path

<Tabs syncKey="quickstart-path">
  <TabItem label="Try it locally">
    Best for development or testing on your machine. **No `.env` file required.**

    From the [App repo](https://github.com/SyncBins/App) root:

    ```sh
    npm run dev:compose
    ```

    Open **http://localhost:5173**. The UI proxies `/api` and `/ws` to the server container. Data persists under `App/server/data/` (SQLite + `blobs/`).

    Stop with `Ctrl+C` or `npm run dev:compose:down`.

    <Aside type="note">
    This uses `docker/docker-compose.dev.yml` with `STORAGE_KIND=local` implicitly (server default) and `TENANT_MODE=single`.
    </Aside>

    <Aside type="tip" title="Phone on Wi‑Fi?">
    `http://192.168.x.x` is not a secure context on iOS — send/crypto can fail. Use [homelab HTTPS](/self-hosting/tls/#local-network-homelab-https) (`mkcert` + `docker-compose.dev-tls.yml`).
    </Aside>
  </TabItem>

  <TabItem label="Production (your domain)">
    One compose stack: SyncBins + Caddy for Let's Encrypt TLS.

    ### Prerequisites

    - A VPS, homelab PC, or **[Raspberry Pi 4/5](/self-hosting/raspberry-pi/)** (64-bit OS) with Docker
    - A domain (e.g. `box.yourdomain.com`) pointing at the box — or [homelab HTTPS](/self-hosting/tls/#local-network-homelab-https) on a LAN
    - Ports 80/443 open for ACME (public domain) or mkcert (LAN)

    ### Steps

    <Steps>

    1. **Get the compose bundle** — either download `syncbins-deploy-*.zip` from [GitHub Releases](https://github.com/SyncBins/App/releases) (recommended on a Pi), or clone the App repo:

       ```sh
       git clone https://github.com/SyncBins/App.git
       cd App/docker
       cp .env.example .env
       ```

       Release zips include `docker-compose.yml`, `Caddyfile`, and `.env.example` ready to run.

    2. **Set two required values** in `.env`:

       ```ini title="docker/.env"
       DOMAIN=box.yourdomain.com
       BLOB_URL_SECRET=…   # openssl rand -hex 32
       ```

       Leave `STORAGE_KIND=local` unless you want [cloud blobs](/self-hosting/storage/).

    3. **Start the stack:**

       ```sh
       docker compose up -d
       docker compose logs -f syncbins
       ```

    4. **Open your URL** and complete onboarding — same wizard as hosted syncbins.com. The **first device** to finish setup becomes the bootstrap device (generates the recovery phrase and master key).

       <Aside type="caution" title="Bootstrap immediately">
       Until bootstrap completes, anyone who can reach the URL could claim the server. Run onboarding yourself right after first startup.
       </Aside>

    5. **Verify health:**

       - `curl -s https://box.yourdomain.com/api/health` → `{"ok":true,"version":"0.0.0"}`
       - Settings → Server: green status, latency shown
       - `./data/syncbins.db` and `./data/blobs/` exist on the host

    </Steps>

    Compose files live in `App/docker/` (`docker-compose.yml`, `Caddyfile`, `.env.example`). See the [App docker README](https://github.com/SyncBins/App/tree/main/docker) for bind-mount layout.
  </TabItem>
</Tabs>

## What you don't need to configure

For a standard **single-user self-hosted** box:

| Topic | Default | Action |
|-------|---------|--------|
| Storage | Local disk (`./data/blobs`) | Nothing — works out of the box |
| Tenancy | Single tenant | Keep `TENANT_MODE=single` |
| Stripe / billing | Off | Ignore Stripe env vars |
| R2 / Azure | Off | Only if you opt into cloud blobs |

You are **not** expected to paste a wall of env vars into a Settings screen. Production needs **domain + secret**; cloud storage is an optional appendix.

## Optional: cloud blob storage

If local disk isn't enough, set `STORAGE_KIND=r2` or `STORAGE_KIND=azure` and the credentials from [Choosing storage](/self-hosting/storage/). Restart the container. No UI change required.
